# 写作方法自进化

## 为什么需要这个 Skill

不管方法论写得多细，AI 写出来的东西总和青城自己写的有差距。最有价值的训练数据是「AI 原稿 → 青城改完的终稿」这个 diff——青城每次改 AI 稿的动作里包含了他的判断标准和偏好，比任何方法论文档都精确。如果不把每次改稿沉淀为规律，改完就丢了。

参考来源：叁斤在 3.28 航海家AI大会分享的"把人改 AI 的动作做成数据，反哺方法论"。

## 触发条件

- 青城说「写作方法自进化」「自我进化」「学我的写作方式」「学我的修改方式」「积累修改规律」
- 周复盘时顺带检查是否有至少 3 对新的可对比文章
- 改完 3-5 篇后就值得跑一轮

## 前置条件

每个高频内容账号的目录结构：

```
03-项目管理/{项目}/
├── 待审核/    ← AI 写完先放这里（原稿不动，是训练集）
└── 已发布/    ← 青城改完发的版本放这里
```

两份同名文件就是一对训练样本。至少凑齐 3 对才执行——少于 3 对找不出共性规律。

## 执行流程

### 步骤 1：扫描同名对

扫描所有 `03-项目管理/` 下各项目的 `待审核/` 和 `已发布/` 目录，找出文件名相同（或高度相似）的文件对。

### 步骤 2：门控检查

同名对少于 3 对 → 报告"样本不足，当前 N 对，需要至少 3 对"并退出。≥3 对 → 继续。

### 步骤 3：逐对 diff

逐对对比 AI 原稿和青城终稿，标注每处修改的类型：

- **删**：青城删掉的内容
- **替**：青城替换的表述
- **加**：青城新增的内容
- **重排**：青城调整的结构/顺序
- **改语气**：青城调整的语气/态度

### 步骤 4：跨对找共性

跨至少 2 对出现的相同修改模式 → 立为规律。单次出现的修改不立规律（统计学不允许从 n=1 推断）。

### 步骤 5：写入修改规律

规律按账号类型归类，写入 `04-内容生产/方法论/{对应子类}/修改规律.md`。子类对应关系：内容创作 / 营销销售 / 搜索 SEO 等，根据项目所属账号判断。

### 步骤 6：下次生效

写入后，AI 下次为该账号写作时自动读取对应 `修改规律.md` 并遵守。

## 升级机制

某条规律在 **3 轮进化里反复出现** → 状态从 `观察中` 升级为 `已固化` → 提请青城确认是否将该规律从「修改规律」升级到「方法论正文」。

## 输出格式

写入的每条规律使用以下格式：

```markdown
### 规律：[一句话描述]
- **状态**：观察中 / 已固化
- **出现次数**：N/3 轮
- **修改类型**：删/替/加/重排/改语气
- **典型案例**：原文 → 改后
- **适用场景**：[什么时候用这条规律]
```

## 约束

- **不自动修改方法论正文**：只能写到「修改规律」单独区域，升级到正文须经青城确认
- **不删除原稿**：`待审核/` 里的 AI 原稿是训练集，永远不删
- **不从单次修改总结规律**：至少 2 对重复才立规律
- **本 Skill 是跨所有写作子类的元机制**：产出的修改规律按账号类型分流到对应子方法论下

## 位置说明

物理归在 `04-内容生产/方法论/` 顶层（与子方法论平级），通过软链接 `.claude/skills/写作方法自进化` → 本目录。本文件即唯一真实文件。
