# 周复盘

> 版本：v1.0 · 2026-06-11

每周轻量复盘工作流。把过去 7 天的知识库增量、链接缺口、MOC 变动、青城认知更新整合成一份结构化周记，让 AI 和青城对本周演进有共同认知。

## 触发条件

- 青城说「周复盘」「做周复盘」「本周复盘」
- 周一 SessionStart hook 自检触发（自动提醒，不等青城说）
- 周末（周日）或青城问起「这周做了什么」「最近怎么样」时主动提议

## 执行步骤

### 1. 扫描本周新增卡片

统计 `02-知识库/` 中 `created` 在过去 7 天内的卡片，列出文件名 + status + source_type。

### 2. 扫描链接缺口

找出以下卡片：
- 没有「相关链接」段的卡片
- 相关链接少于 2 条的卡片
- typed link 类型缺失（应有支撑/应用于/张力/互见，缺失则标记）

### 3. 检查 MOC 覆盖

对比本周新增卡片和现有 MOC，找出还没有被任何 MOC 引用的新卡片。提议归入哪个 MOC。

### 4. 同步 01-关于我

扫描本周对话存档，判断是否有值得写入 `01-关于我/` 的个人认知更新。如有，列出待并入项。

### 5. 调用记录健康度

读取根目录 `00-运转日志/知识卡片调用记录.md` 的「调用记录」表：
- 本周新增调用条数（健康频率：5-20 条）
- 高频被调用的卡片（top 3）
- 是否有断档日（连续 ≥2 天无新调用）

### 6. MOC 生长分布检查

逐 MOC 检查本周新增卡片数，输出分布表：

| MOC | 本周新增 | 最近调用 | 活跃度 |
|-----|---------|---------|--------|
| MOC-XXX | N | YYYY-MM-DD | 🟢/🟡 |

- 本周零增长的 MOC → 标 🟡，提示「该领域本周无新增，是否注意力收窄或该 MOC 已完成使命？」
- 连续 30 天零增长 + 零调用 → 标 🔴，提示「该 MOC 是否该合并/归档/删除」

MOC 健康度与知识卡片生命周期逻辑一致——都是「被使用才健康」。连续无活动的 MOC 不等于该删，但值得青城看一眼。

### 7. 调用记录定性补充

在上一步定量统计的基础上做定性判断：**从本周对话存档正文中识别虽调用记录 count 不高、但在对话里被反复提及的卡片**。这些卡片的实际使用频率高于调用记录反映的数值，可能需要升级 status 或在 MOC 中提升权重。

执行方式：两条路径并行——① 扫描本周对话存档 AI 摘要中的「核心引用卡片」字段，汇总出现频次；② grep 对话存档正文中 `[[卡片名]]` 出现次数。两者与调用记录 count 对比。出现次数 > count 的卡片列为「被低估卡片」。

### 8. 生成周记文件

输出到 `00-运转日志/周复盘/YYYY-Www.md`（W 代表 ISO 周数）。

结构：

```markdown
# 周复盘 · YYYY 第 ww 周（MM-DD ~ MM-DD）

## 本周知识库增量
[新增 N 张卡片 / 说明书更新 X 项 / MOC 变动]

## 链接缺口
[需要补的链接清单]

## MOC 归位
[本周新卡片的 MOC 归入提议]

## 个人认知更新
[01-关于我/ 待并入项]

## 调用记录健康度
[频次评估 + top 卡片 + 被低估卡片清单]

## MOC 生长分布
[MOC 本周新增分布表 + 零增长 MOC 提示 + 30 天无活动 MOC 警告]

## 下周建议
[3 句话给青城：优先做什么、避开什么、关注什么]
```

### 9. 错字扫描（铁律）

写入完成后必须 grep 扫描新增文件的可疑字符。

## 异常处理

| 场景 | 触发条件 | 处理动作 |
|------|---------|---------|
| 本周无新增卡片 | `created` 在过去 7 天的卡片数为 0 | 不报错退出，周记中注明「本周无新增卡片」，正常输出其他章节 |
| 本周无对话存档 | 过去 7 天无新的对话记录文件 | 跳过「个人认知更新」和「调用记录定性补充」，周记中注明 |
| 调用记录文件缺失 | `00-运转日志/知识卡片调用记录.md` 不存在 | 跳过调用记录步骤，周记末尾提醒「调用记录文件缺失，请检查 cron 是否运行」 |
| grep 扫描结果为空 | 某些检查项无匹配 | 如实记录「0 项」，不编造数据 |
| 周复盘文件已存在 | 同周文件已写过 | 追加「补充复盘」段而不是覆盖，标注追加时间 |
| MOC 目录为空 | 10 个 MOC 全无变动 | 正常输出「本周零增长」表格，逐 MOC 标 🟢（无增长但正常） |

## 反例清单（不要做的事）

| # | 反模式 | 为什么不要做 | 替代做法 |
|---|--------|-------------|---------|
| 1 | **周复盘写成月度体检** | 深入扫全库链接/元数据/生命周期是体检的事，周复盘做了就越界 | 只看增量（本周新增+本周变动），深度交给体检 |
| 2 | **编造数据填满表格** | 本周真的没新增就写 0，强行凑数据等于骗青城 | 如实记录，零增长也是信号 |
| 3 | **下周建议写废话** | 「继续努力」「保持节奏」没有信息量 | 写具体行动：「先修复 N 条断链」「本周 X 方向零增长，下周重点关注」 |
| 4 | **跳过 MOC 活跃度检查** | 零增长 MOC 是注意力收窄的信号，跳过等于漏掉重要趋势 | 每个 MOC 必须过一遍 |
| 5 | **忽略「被低估卡片」** | 有些卡片在对话里反复引用但 count 低，不打标记就永远是 dormant | 两条路径（摘要+正文 grep）对比，列出被低估卡片 |

## 铁律

- **触发即跑**：青城说「周复盘」就立即执行，不要反问
- **轻量优先**：复盘不是体检，扫描动作要快，3-5 分钟出周记
- **下周建议要具体**：不写"继续努力"，写"先做 X 再做 Y"
- **不重复月度体检**：周复盘只看增量与短期趋势，深度健康度交给月度体检
- **中文写作，中文标点**
