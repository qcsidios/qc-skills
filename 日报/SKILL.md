# 日报

每天早上 6 点之后启动 Claude Code 时，**hook 自检 + AI 直接执行 + 幂等跳过**：昨日日报不存在就生成，存在就跳过。一天开多少个终端窗口与此无关——只看文件是否存在。

## 为什么是这个流程

日报和对话存档职责切干净：
- **对话存档**：单日的过程、决策、认知、隐线——「这一天具体发生了什么、想什么」
- **日报**：跨日才有意义的快照和趋势——「这一段时间在演化什么」

历史教训：日报曾长期复述对话存档里的关键决策、认知变化、对话模式观察、隐线，重叠率超过 70%，违反「不重复对话存档」铁律。2026-05-23 重构后日报瘦身为只保留跨日不可替代的内容。

## 报告流程

### 第一步：幂等自检（由 hook 完成，AI 直接接收指令）

**`.claude/hooks/session-start.sh` 在 6 点之后启动时自动跑这套自检**，AI 收到 hook 注入的"待执行任务"上下文就直接执行，**不要再询问青城是否触发**。

判断逻辑（已写进 hook，这里记录原理）：
- `ls 00-对话存档/日报/${TODAY}*.md` 通配匹配（**TODAY 前缀，因为命名规则是"前缀=启动当天，内容覆盖昨天"**——别用 YESTERDAY 做前缀，那是昨天写的覆盖前天的日报）
- 匹配到任意文件（无论主题如何） → 视为已生成，**静默跳过**
- 未匹配到 → AI **直接生成**

**绝对不能用「这是今天开的第几个终端」当判断依据**——这与是否需要生成日报无关。

**「日」以早上 6 点为界**：6 点之前的对话属于前一天（含凌晨/熬夜），6 点之后属于新一天。这是与对话存档 Skill 共用的时间约定。当前时间 < 06:00 时 hook 不会触发，AI 不要主动跑。

**命名规则（关键，不要弄错）**：
- 文件名格式 = `YYYY-MM-DD-日报-主题关键词.md`（如 `2026-05-23-日报-从体检到v1.2与基础设施收口.md`）
- 日期 = 启动当天的日期（5/23 第一次启动 → 文件名前缀 `2026-05-23`）
- **主题关键词来源**：直接复用昨日对话存档的主题关键词（日报覆盖前一天，所以与前一天对话存档同主题）。如果存在多个对话存档，挑能概括全天的那个；如果对话存档主题已包含日期，剥掉日期部分
- 内容 = 总结前一天（5/23 第一次启动生成的日报 → 内容覆盖 5/22）
- 内部标题 = 文件名同日期（`# 日报 · 2026年5月23日`），副标题写「昨日（5月22日）跨日状态快照」
- 即「今天回看昨天」——文件归档到「今天」这个时间锚点上，内容是对昨天的快照

### 第二步：收集昨日信息

扫描以下数据源：

- 知识库卡片数量统计（`02-知识库/` 下所有 .md 文件数，排除 MOC/index/log）
- 个人说明书文件数（`01-关于我/` 下实质文件，不含 CLAUDE.md 子宪法）
- 对话存档与日报数量（`00-对话存档/对话记录/` 与 `00-对话存档/日报/`）
- 待处理素材（`素材箱/_待处理/` + `Clippings/`）
- `02-知识库/log.md` 中昨日的变更记录
- `git log` 当天提交（看跨日趋势）
- 必要时翻昨日对话存档的「全天产出清单」节，但**不要复述其结论**——只用于核对增量数字

### 第三步：撰写报告

新骨架（短小为荣，长则失职）：

```markdown
---
type: 日报
date_covered: YYYY-MM-DD
summary: [1-2 句、**不超过 200 字**，概括昨日核心变化，供日报看板卡片渲染。要点：具体名词 + 动词，不写感叹和评价。例：「协议三连升 v1.0→v1.1→v1.2，铁律精简，49 个 Skill 跨工具齐整。」]
---

# 日报 · YYYY年MM月DD日

> 昨日（MM月DD日）一句话定调：[一句话概括昨日在做什么 / 处于什么阶段]。

## 一、知识库状态

| 指标 | 数值 | 较前日 |
|------|------|--------|
| 知识卡片总数（02） | N | +/- |
| 个人说明书文件（01） | N | +/- |
| 对话存档 | N 篇 | +/- |
| 日报 | N 篇 | +/- |
| 待处理 | N | +/- |

[可选：一句话备注增量来源，如"5 张新卡均来自 X 对话回流"]

## 二、本周演化轨迹

| 日期 | 主题 | 强度 |
|------|------|------|
| MM/DD | ... | 极低/低/中/高/极高 |
| ... | ... | ... |
| MM/DD | [昨日] | ... |

[最多 7 行，覆盖昨日往前 7 天。新日子从知识库还没启动的那天起，不强行凑]

## 三、跨日提示（仅在有显著趋势时写）

[一两句话。例：连续两天极高强度后通常需要喘息；某项基础设施改动可能在 N 天后才显效；某条铁律 X 天未触发可能要看是否仍需要等]

[没显著趋势 → 整节跳过，不要硬写]
```

**frontmatter 字段说明**：
- `type: 日报`：固定值，用于看板筛选
- `date_covered`：实际覆盖的那一天日期（不是写日报的当天）
- `summary`：1-2 句详细总结，**给日报看板渲染用**。与正文「一句话定调」互补——定调给方向感（standfirst 风格），summary 给实质内容（具体名词 + 动词）。两者不要互相重复

### 第四步：写入

文件名：`YYYY-MM-DD-日报-主题关键词.md`，写入 `00-对话存档/日报/`。

### 第五步：错字扫描（铁律）

写入完成后，必须 grep 扫描新增文件的可疑字符。重点排查：辨证/辩证、锚点/锡点、沉淀/沉澳、祛魅/祠魅、一卒/一卆、炫耀/炸耀，以及其他低频汉字同音/形近替换。

## 铁律

- **6 点后启动 hook 触发即做**：收到 hook 注入的"待执行：YYYY-MM-DD 日报未生成"指令就**直接生成**，不要先问青城是否需要；同时收到对话整理修订指令时按正确顺序串联——**先整理修订前一日对话，再出今日日报**（日报需要基于整理好的对话做跨日总结，反过来就只能基于碎片）
- **唯一幂等依据是文件存在**：`ls ${TODAY}*.md` 匹配到 → 跳过；否则补（**前缀=启动当天 TODAY，不是 YESTERDAY**）
- **不重复对话存档**：日报只写「跨日才有意义」的内容（增量数据、本周轨迹、跨日趋势提示）。关键决策、认知变化、对话模式观察、隐线、待办——全部归对话存档管，日报不复述
- **短小为荣**：好的日报应当 30-60 行结束。如果超过 80 行，说明在复述对话存档
- **跨日提示按需写**：没显著趋势就跳过整节，不要硬凑
- **中文写作，中文标点**
