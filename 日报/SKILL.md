# 日报

> 版本：v1.1 · 2026-06-15

每天早上 6 点之后启动 Claude Code 时，**hook 自检 + AI 直接执行 + 幂等跳过**：昨日日报不存在就生成，存在就跳过。一天开多少个终端窗口与此无关——只看文件是否存在。

## 为什么是这个流程

日报和对话存档职责切干净：
- **对话存档**：单日的过程、决策、认知、隐线——「这一天具体发生了什么、想什么」
- **日报**：跨日才有意义的快照和趋势——「这一段时间在演化什么」

历史教训：日报曾长期复述对话存档里的关键决策、认知变化、对话模式观察、隐线，重叠率超过 70%，违反「不重复对话存档」铁律。2026-05-23 重构后日报瘦身为只保留跨日不可替代的内容。

## 报告流程

### 第一步：幂等自检（由 hook 完成，AI 直接接收指令）

> 具体实现见 `.claude/hooks/session-start.sh`，此处记录原理供 AI 理解。

**hook 在 6 点之后启动时自动跑这套自检**，AI 收到 hook 注入的"待执行任务"上下文就直接执行，**不要再询问青城是否触发**。

判断逻辑：
- `ls 00-运转日志/日报/${TODAY}*.md` 通配匹配（**TODAY 前缀——命名规则是"前缀=启动当天，内容覆盖昨天"，别用 YESTERDAY 做前缀**）
- 匹配到任意文件（无论主题如何） → 视为已生成，**静默跳过**
- 未匹配到 → AI **直接生成**

**绝对不能用「这是今天开的第几个终端」当判断依据**——这与是否需要生成日报无关。

**「日」以早上 6 点为界**：与对话存档 Skill 共用。当前时间 < 06:00 时 hook 不会触发，AI 不要主动跑。

**命名规则**：
- 文件名 = `YYYY-MM-DD-日报-主题关键词.md`，日期 = 启动当天
- 主题关键词直接复用昨日对话存档的主题关键词（日报覆盖前一天，同主题）
- 内容覆盖前一天（5/23 启动 → 内容覆盖 5/22）——「今天回看昨天」

### 第二步：收集昨日信息

扫描以下数据源：

- 知识库卡片数量统计（`02-知识库/` 下 .md 文件数，排除 MOC/index/log）
- 个人说明书文件数（`01-关于我/` 下实质文件，不含 CLAUDE.md 子宪法）
- 对话存档与日报数量（`00-运转日志/对话记录/` 与 `00-运转日志/日报/`）
- 待处理素材（`素材箱/_待处理/` + `素材箱/文章收藏/` 新增文件）
- `02-知识库/log.md` 中昨日的变更记录
- `git log` 当天提交（看跨日趋势）
- 必要时翻昨日对话存档的「全天产出清单」节，但**不要复述其结论**——只用于核对增量数字

### 第三步：撰写报告

新骨架（短小为荣，长则失职）：

```markdown
---
type: 日报
date_covered: YYYY-MM-DD
summary: [1-2 句、不超过 200 字，概括昨日核心变化。要点：具体名词 + 动词，不写感叹和评价。]
---

# 日报 · YYYY年MM月DD日

> 昨日（MM月DD日）一句话定调：[一句话概括昨日在做什么 / 处于什么阶段]。

## 一、知识库状态

| 指标 | 数值 | 较前日 |
|------|------|--------|
| 知识卡片总数（02） | N | +/- |
| 个人说明书文件（01） | N | +/- |
| 对话存档 | N 篇 | +/- |
| 日报 | N 篇 | +/- |
| 待处理 | N | +/- |

[可选：一句话备注增量来源]

## 二、本周演化轨迹

| 日期 | 主题 | 强度 |
|------|------|------|
| MM/DD | ... | 极低/低/中/高/极高 |
| MM/DD | [昨日] | ... |

[最多 7 行，覆盖昨日往前 7 天。]

## 三、跨日提示（仅在有显著趋势时写）

[没显著趋势 → 整节跳过，不要硬写]
```

**frontmatter 字段说明**：
- `type: 日报`：固定值，用于看板筛选
- `date_covered`：实际覆盖的那一天日期（不是写日报的当天）
- `summary`：给日报看板渲染用。与正文「一句话定调」互补——定调给方向感，summary 给实质内容

### 第四步：写入

文件名：`YYYY-MM-DD-日报-主题关键词.md`，写入 `00-运转日志/日报/`。

### 第五步：错字扫描（铁律）

写入完成后 grep 扫描新增文件的可疑字符。重点排查：辨证/辩证、锚点/锡点、沉淀/沉澳、祛魅/祠魅、一卒/一卆、炫耀/炸耀，以及其他低频汉字同音/形近替换。

## 反例清单

| # | 反模式 | 替代做法 |
|---|--------|---------|
| 1 | 日报复述对话存档里的关键决策、认知变化 | 只写跨日数据：增量、轨迹、趋势提示 |
| 2 | 日报超过 80 行 | 30-60 行结束。超过 = 在复述对话存档 |
| 3 | 问青城「要不要生成日报」 | 收到 hook 指令直接生成，不等确认 |
| 4 | 用「启动了几个终端」判断是否需要生成 | 只看文件是否存在，不看过往终端窗口数 |
| 5 | 跨日提示硬凑——没趋势也硬写 | 没显著趋势就跳过整节 |

## 铁律

- **hook 触发即做**：收到 hook 注入的指令就直接生成，不问青城。同时收到对话整理修订指令时**先整理修订前一日对话，再出今日日报**
- **唯一幂等依据是文件存在**：`ls ${TODAY}*.md` 匹配到 → 跳过。前缀=启动当天 TODAY，不是 YESTERDAY
- **不重复对话存档**：日报只写跨日数据（增量、轨迹、趋势）。关键决策/认知/隐线/待办归对话存档
- **短小为荣**：30-60 行。超过 80 行 = 在复述对话存档
- **跨日提示按需写**：没趋势就跳过，不硬凑
- **中文写作，中文标点**
