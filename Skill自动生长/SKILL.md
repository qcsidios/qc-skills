# Skill 自动生长

> 版本：v1.0 · 2026-06-07

AI 的肌肉记忆不该停留在一次次手动重复。当同一个知识库操作被重复执行 3 次时，说明这个操作已经有稳定的模式和价值——应该固化为 Skill。

## 流程

### 第一步：识别可生长操作并更新计数器

AI 在每次执行操作后，扫描 `AI协作工具/操作重复计数.md` 判断是否命中已有记录：

- **命中** → 次数 +1，更新「最近日期」
- **新操作** → 追加一行 `| 操作描述 | 1 | 日期 |`

以下信号触发记录：

- 同一个多步骤流程反复执行
- 同一组 bash 命令反复敲
- 同一个判断/决策逻辑反复用
- 同一个文件操作模式反复做

**不触发记录的**：单次简单命令（`ls`、`git status`）、对话中自然发生的问答、已经被 Skill 覆盖的操作。

计数器是文件系统级的——不依赖 AI 的跨会话记忆。AI 每次执行操作时扫一眼这个文件，命中就 +1。

### 第二步：第 3 次执行时告知青城

当同一个操作累计到第 3 次时，AI 主动告知青城：

> 「『XXX 操作』已经做了 3 次了，要不要固化为一个 Skill？这样以后直接调用就行。」

附带简要说明：这个操作做什么、为什么值得固化、放在哪个分类。

### 第三步：青城确认后写入

青城同意后，AI 执行：

1. **确定 Skill 位置**：
   - 协作机制类（操作方法/流程规范）→ `AI协作工具/skills/{名称}/`
   - 业务领域类 → 按职能归位（如 `03-内容生产/方法论/`、`04-认知工坊/` 等）
   
2. **调用 `skill-creator` 编写 SKILL.md**：不手写，通过 skill-creator 按标准格式创建并审查

3. **创建软链接**：
   ```bash
   ln -s ../../AI协作工具/skills/{名称} .claude/skills/{名称}
   ```

4. **更新协作系统**：
   - 根 CLAUDE.md 变更记录追加一行
   - `02-知识库/log.md` 记录新 Skill 注册

## 铁律

- **3 次门槛不放松**：不到 3 次不提议——单次操作可能是偶然，两次可能还看不清模式，三次确认不是巧合。计数器以 `AI协作工具/操作重复计数.md` 为唯一依据，不靠记忆
- **必须经青城确认**：不自行写入 Skill
- **简单优先**：如果青城说「不需要」，就继续手动执行，不勉强
- **Skill 随用随进化**：固化后不是终点，执行中发现可改进点就更新（见协作原则第 10 条）
