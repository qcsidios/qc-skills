# 知识库体检

> 版本：v2.2 · 2026-06-20
> v2.1→v2.2：**来源链验证 + 阈值校准 + 时效修正 + typed link 语义审计**。① 第五步新增来源链接可解析性验证；② 六维定量扫描新增「来源链接可解析率 ≥ 90%」维度；③ 过时表述列表扩充；④ MOC 覆盖率健康线 70%→90%→100%；⑤ 铁律「5 分钟」改为分段；⑥ 触发条件建议化；⑦ 第四步新增 typed link 类型合规扫描（四标准类型）和语义合理性抽检（张力链接人工判断）。

> 当青城说「体检」「健康检查」「扫一下知识库」等时调用。触发方式：① 青城手动；② 周复盘时 AI 主动提议。

定期扫描整个知识库，识别需要维护的问题，输出体检报告供青城决策。

## 为什么需要体检

知识库是地基和数据基建。新卡片在加，旧卡片可能链接断了、与新理解矛盾、长期不被使用。没有定期体检，地基会从「数据基建」变成「垃圾场」。

体检是**诊断**，不是清理。提出问题，由青城决定怎么处理。

## 触发

- 青城手动触发（说「体检」「健康检查」等）
- 周复盘时 AI 主动提议
- 大批量编译素材后（一次入库 10+ 张卡片后）→ 建议体检，非强制

## 体检流程

### 第一步：基础扫描

统计卡片总数、MOC 数、最近 7 天新增。逐个检查每张卡片的 YAML 头（created + source_type）、`**来源**` 标注、文件名与 H1 标题一致。MOC 覆盖率、死链等快速拓扑由第一阶段（知识网络快速定量检测，见文末）覆盖，本步引用其输出不重复执行。

### 第二步：网络健康

**链接深度**：
- 区分来源链接（`**来源**：[[xxx]]`）与概念链接——来源行不计入概念入度，避免素材源文件冒充骨架卡片
- 同领域 Top 10 入度骨架卡片之间互链检查——缺链是网络稀疏信号

**生命周期状态**（读 `00-运转日志/知识卡片调用记录.md`）：
- `active`：调用 ≥ 1
- `dormant`：调用 = 0 且入度 = 0
- `candidate_for_review`：调用 = 0、入度 = 0、创建 > 90 天——候选删除需青城确认
- 调用记录为空时回退用 `file.mtime + 入度` 估算

**evergreen 静默检查**：evergreen 超过 90 天未修改且调用 = 0 → 列「静默清单」，不擅自降级，青城回访后判断。

### 第三步：生命周期升降级

三级生长体系（seed → growing → evergreen），由体检自动执行：

| 升级 | 触发条件 |
|------|---------|
| seed → growing | 调用 ≥ 3 或概念入度 ≥ 2 或多来源交叉验证（入链来自 ≥2 种 source_type） |
| growing → evergreen | 调用 ≥ 10 或概念入度 ≥ 5 |

**入度计算**：排除 `**来源**` 行和 `MOC/` 目录下的引用。降级规则：evergreen 90 天调用 = 0 → growing；growing 调用 = 0 且入度 = 0 → seed。

执行：逐张判断 → 列出清单 → 自动改 frontmatter `status` → log.md 追加记录。

### 第四步：语义深度扫描

**冗余检测**：文件名 bigram Jaccard ≥ 0.40 或核心短语重叠 ≥ 5 字的卡片对，列候选。不擅自合并，逐对让青城判断。

**MOC 内互链密度**：实际边数 / n×(n-1)/2。≥15% 🟢，10-15% 🟡，<10% 🔴。

**01↔02 跨区桥梁**：逐 01 文件查裸文本中出现的 02 卡片名→建议补链；反向扫 02→01 缺口。覆盖率 ≥ 80% 健康。

**0 入度卡片**：列清单 + 建议回链的高入度骨架卡片。事件性卡片天然零入度合理。

**张力卡片边界**：维护已知张力对清单，审查互链和显式边界标注。缺边界 → 建议补。

**骨架卡片互链**：Top 10 入度骨架之间缺链对清单。

**Typed link 审计**：调用「typed link审计」Skill，执行全量类型合规扫描 + 张力/支撑语义抽检。本步引用其输出报告，不重复执行。

### 第五步：系统一致性

- **索引与 MOC**：每卡至少被一个 MOC 收录。index.md 提及的 MOC 是否存在。是否有重复条目
- **Skill 软链接**：`.claude/skills/` 下软链接 → 验证目标文件存在。检查 `AI协作工具/skills/` 下有无 SKILL.md 但缺少对应软链接
- **过时表述**：grep 已废弃概念（如「外脑」「认知报告」「剪存文档」→ 应为「地基和数据基建」「剪存」），以及旧视频日记文件名格式（缺少日期前缀的命名）。与当前命名约定对齐
- **来源链接可解析性**：逐卡验证 `**来源**` 中引用的素材箱文件名是否在素材箱目录下真实存在。来源死链是知识可追溯性的致命伤——不知道一张卡从哪来的，等于这张卡不可审计。健康线：来源可解析率 ≥ 90%
- **素材箱**：`_待处理/` 是否为空
- **版本号一致性**：根 CLAUDE.md 版本号与子宪法引用是否对齐

### 第六步：报告与修复

**输出报告**：`00-运转日志/体检报告/YYYY-MM-DD-体检报告.md`

文件格式要求（看板抓取契约）：标题 `## 总体健康度` 后用单行 🟢/🟡/🔴 emoji 开头的总结，文件名 `YYYY-MM-DD-(主题).md`。

报告结构：总体健康度 → 基础统计 → 需要决策的事项（🔴/🟡/🟢） → 健康指标表 → 本月认知叙事（≤200 字） → 下次体检建议。

**修复**：🔴 项默认同日修复（除非青城说等或需要他语义判断）。按 🔴→🟡→🟢 顺序。修复后**必须回写报告**：健康指标表加「修复后」列 + 新增「修复结果」章节 + 重跑机械扫描验证指标。诊断后必修复，修复后必回写——下次体检的基线是上次的「修复后」列。

## 反例清单

| # | 反模式 | 替代做法 |
|---|--------|---------|
| 1 | 体检报告出了但 🔴 项不修——「下次再说」 | 🔴 项同日修复，不等 |
| 2 | 修复完不回写报告——「已经修了就行」 | 必须回写「修复后」列，下次基线靠它 |
| 3 | 擅自动语义判断项——自己决定合并/重命名 | 列候选让青城裁决 |
| 4 | 把来源链接当概念入度——素材源文件冒充骨架卡片 | 来源行不计入概念入度 |
| 5 | 体检报告写成技术审计文——堆数据不给判断 | 每条说清「什么问题、怎么修、修了之后什么样」 |

---

## 铁律

- **诊断后必修复，修复后必回写**：下次体检基线 = 上次「修复后」列
- **简单优先**：快速定量扫描 5 分钟内完成；完整体检按需深入，不设硬性时间上限
- **可读优先**：报告给青城看，每条说清问题+建议+修复结果
- **修复不擅动语义项**：合并/重命名/「该不该回链」必须列候选；机械补链/补字段可直接做

# 附录：知识网络快速定量检测

> 原独立 Skill 已合并至此，作为体检第一阶段的机械扫描层。每次体检先跑本段，5 分钟内完成。

## 触发

- 知识库卡片总数 ≥ 上次检测基数 + 10 时自动触发
- 体检联动：体检流程第一步

## 六维扫描

| 维度 | 健康线 |
|------|:------:|
| 0 入度卡片占比 | < 20% |
| 0 出度卡片占比 | < 15% |
| 入度 ≥ 3 的桥梁卡片 | ≥ 5% |
| 出入度均为 0 的孤立节点 | < 5% |
| MOC 覆盖率 | 100% |
| typed link 总数 | ≥ 卡片数 × 0.5 |
| 来源链接可解析率 | ≥ 90% |

定量检测通过 → 跳过体检大部分机械扫描。不通过 → 进入完整体检。结果作为健康指标表基线。
